# Visual and research sources

## Visual policy for emotional planet pages

The emotional detail pages no longer render the generated editorial SVG image set in `frontend/assets/images/planets/`.

The visible visual system now uses:

- Real planet texture files from `frontend/planets/`, cropped into circles for UI only.
- HTML/CSS educational diagrams for body signals, emotion loops, and intervention steps.
- Text labels derived from each planet's own emotion data.

This keeps every visual explanatory and editable without generating a new bitmap. The old SVG files remain archived for now so existing user assets are not deleted without approval.

## Solar-system boundary

Planet texture files are allowed only in UI components such as detail-page heroes, menu icons, information panels, and footer navigation. They must not replace or override the procedural orbiting planets in the home-page solar-system canvas.
